# Web Fundamentals for VAPT

This document covers the foundational concepts I learned before starting practical work in Vulnerability Assessment and Penetration Testing (VAPT). These fundamentals help in understanding how web applications function and how security vulnerabilities arise.

---

## 🌐 Client-Server Model

- A **client** is a device or application that sends requests (e.g., browser)
- A **server** is a system that processes requests and sends responses
- Communication happens over the internet using protocols like HTTP/HTTPS

---

## 🌍 Domain & DNS

- A **domain** is a human-readable address (e.g., example.com)
- **DNS (Domain Name System)** converts domain names into IP addresses

### Types of DNS Servers:
- Root Server  
- Primary Server  
- Secondary Server  

### How DNS Works:
1. User enters a domain
2. DNS resolves it into an IP address
3. Request is sent to the server

---

## 🧾 IP Address

- A unique identifier for devices on a network
- Used to locate and communicate with servers

---

## 🔐 HTTP & HTTPS

- **HTTP (HyperText Transfer Protocol)** → not secure  
- **HTTPS** → secure version using encryption  

### SSL/TLS:
- Protocols used to encrypt communication between client and server

---

## 📡 HTTP Methods

- **GET** → Retrieve data  
- **POST** → Send data  
- **PUT** → Update data  
- **DELETE** → Remove data  

---

## 📊 HTTP Status Codes

- **1xx (Informational)** → Request received, continuing process  
- **2xx (Success)** → Request successful (e.g., 200 OK)  
- **3xx (Redirection)** → Further action needed (e.g., 301 Moved Permanently)  
- **4xx (Client Errors)** → Issue from client side (e.g., 404 Not Found)  
- **5xx (Server Errors)** → Issue from server side (e.g., 500 Internal Server Error)  

---

## 🍪 Cookies & Session ID

- **Cookies** are small pieces of data stored in the user's browser  
  - Used to remember user information (e.g., login, preferences)

- A **Session ID** is a unique identifier generated by the server  
  - It is used to track a user's session on the server  
  - Usually stored inside a cookie  

👉 Example:
When you log in to a website, the server creates a session and sends a session ID to your browser. This allows the server to recognize you in future requests.

---

## 🔑 Tokens

- Used for authentication and authorization  
- Common in modern web applications (e.g., JWT)  
- Instead of sessions, tokens are sent with each request to verify identity  

---

## 🕵️ Reconnaissance

- The process of gathering information about a target  
- Includes identifying:
  - Domains  
  - Subdomains  
  - Technologies used  

---

## 🌐 Domains & Subdomains

- **Domain** → main website (example.com)  
- **Subdomain** → sub-section (admin.example.com)  

---

## 🎯 Summary

Understanding these fundamentals is essential before performing VAPT, as they form the base for identifying, analyzing & exploiting vulnerabilities in web applications.
