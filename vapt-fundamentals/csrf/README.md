# CSRF (Cross-Site Request Forgery)

## 🔍 Overview
In this section, I practiced CSRF attacks in different environments.

The main goal was to understand how an attacker can perform actions on behalf of a logged-in user without their permission.

---

## 🧪 Labs Covered

- **PortSwigger Lab**  
  Performed a simple CSRF attack by modifying request parameters.

- **DVWA**  
  Tested CSRF in low and high security levels to understand how protection works.

- **Real-world web application**  
  Tested similar CSRF behavior on a real-world application for better practical understanding.

---

## 📘 Key Learning

- How CSRF attack works  
- How attackers use authenticated users  
- How requests can be modified and reused  
- Difference between vulnerable and secure applications  

---

## 🛡️ Prevention Awareness

- Use CSRF tokens  
- Validate every request properly  
- Use SameSite cookies  
- Apply proper authentication checks  

---

## ⚠️ Note
This practice was done in a safe lab environment for learning purposes only.
